# 实验日志（Experiment Log）

本日志记录了项目开发过程中的关键参数选择、设计迭代与对比数据。所有实验均在限时两天的探索性开发期间完成，遵循"先框架后细节"的工程哲学。

## 实验 1：故事拆解 Prompt 的演进

**问题**：如何让 LLM 输出 4 个版位风格统一、内容差异化的英文 prompt？

| 版本 | 策略 | 结果 | 问题 |
|---|---|---|---|
| v1 | 直接让 LLM 自由生成 4 个 prompt | 4 张图风格散乱，无法识别为系列 | 缺少风格锚点 |
| v2 | 加入 `style_anchor` 中文字段 | 风格略有改善，但实际生成图仍漂移 | 中文风格描述对图像生成模型作用有限 |
| **v3（最终）** | 用 Qwen-VL 提取**英文风格描述**，强制原文嵌入到 4 个 prompt 末尾 | **4 张图风格高度统一** ✅ | — |

**关键发现**：扩散模型对英文风格描述的响应远好于中文，提示工程必须考虑下游模型的训练语料偏好。

## 实验 2：版位差异化的尺寸策略

**问题**：4 张图如何在尺寸上拉开视觉密度差异？

| 方案 | 前胸 | 后背 | 袖口 | 胸章 | 视觉差异 |
|---|---|---|---|---|---|
| A：统一尺寸 | 1024×1024 | 1024×1024 | 1024×1024 | 1024×1024 | 几乎无差异，4 张图密度相似 |
| **B：差异化（最终）** | **1024×1024** | **720×1280** | **1024×1024** | **1024×1024** | 后背图自然出现叙事感，配合 prompt 中的"信息密度梯度"指令，4 张图密度梯度清晰 ✅ |

**结论**：单靠 prompt 指令难以让模型严格遵守"袖口极简"，**尺寸约束 + prompt 双重作用**才稳定。

## 实验 3：随机种子（Seed）的影响

**问题**：同一 prompt 多次调用，生成图是否一致？

| 配置 | 测试方式 | 结果 |
|---|---|---|
| 不固定种子 | 同一 prompt 调用 3 次 | 3 张图主体相同但细节差异较大（笔触、构图比例不一致） |
| **固定 seed = 42** | 同上 | **3 张图高度一致**，作为"风格一致性的第二重保障" ✅ |

**应用**：在 4 个版位生成时统一使用 `seed=42`，让"内容差异化"的同时"风格保持稳定"。

## 实验 4：风格参考图的有效性测试

**问题**：同一张参考图能否驱动不同故事产出风格一致的作品？

| 输入 | 参考图 | 生成结果 |
|---|---|---|
| 故事 1（反者道之动） | 齐白石虫鱼水墨 | 4 张图均呈现水墨笔触 + 黛黑色调 |
| 故事 2（人能弘道） | 同上 | 同样的水墨语言，但内容呈现"道与术"的二元对比 |
| 故事 3（无身） | 同上 | 同样的水墨语言，禅意主题，独立成系列 |

**结论**：三套作品**共用一张参考图** = 一个完整的"设计 IP 系列"，验证了系统的**跨故事一致性**。这是技术报告 §6.2 的核心论据。

## 实验 5：LLM 输出格式约束

**问题**：LLM 自由文本输出经常缺字段、格式错乱，下游解析容易失败。

| 方案 | 失败率（10 次调用） | 备注 |
|---|---|---|
| 自由文本输出 | 3/10 失败 | 解析需要正则，鲁棒性差 |
| **JSON 结构化输出** `response_format={"type":"json_object"}` | **0/10 失败** ✅ | 强制 LLM 返回合法 JSON |

**应用**：本项目使用 OpenAI 兼容接口的 JSON 模式，保证下游解析的稳定性。

## 实验 6：Mockup 白底融合

**问题**：通义万相生成的图带白色背景，直接贴到 T 恤模板上会形成明显白框。

| 方法 | 效果 | 缺点 |
|---|---|---|
| 不处理 | 白框明显 | ❌ 视觉割裂 |
| **阈值法**（RGB 三通道 > 240 置透明） | 大部分干净融合 ✅ | 复杂背景（浅灰渐变）抠不干净 |
| rembg / SAM 抠图 | 效果最好 | 工程成本高，本项目暂不采用 |

**取舍**：限时框架下选阈值法。技术报告 §8.1 已标注此处的优化方向。
